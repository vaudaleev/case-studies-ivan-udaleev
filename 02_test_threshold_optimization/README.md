# Оптимизация проходного порога теста

## Задача
Подобрать длину теста и проходной порог, чтобы:
- Вероятность прохождения **сильного** кандидата ≥ 80%
- Вероятность отсева **слабого** кандидата ≥ 70%

## Метод
Биномиальное распределение `X ~ Binom(N, p)`  
Расчёт `P(X ≥ threshold)` для сильных (`p=0.98`) и слабых (`p=0.90`) кандидатов.

## Сравнение сценариев
| Сложность | N вопросов | Порог | P(сильный проходит) | P(слабый отсеян) |
|-----------|------------|-------|---------------------|------------------|
| Как на работе | 24 | 23 | 91.7% | 70.8% |
| Проще, чем на работе | 49 | 48 | 91.4% | 71.0% |
| **Сложнее, чем на работе** | **4** | **2** | **91.6%** | **81.9%** |

## Вывод
Оптимальный формат: **короткий тест из 4 вопросов с порогом 2 правильных**.  
- Не отпугивает кандидатов низким барьером входа
- Отсеивает >80% слабых
- Сохраняет >91% сильных в воронке
